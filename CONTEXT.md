## Key design decisions (won't change without a strong reason)
- **Composite GitHub Action**, not Docker — chosen for cold-start speed (~5s vs ~60s)
- **YAML config**, not TOML — matches the format Actions users already write
- **Wikimedia brain anatomy is checked into the repo** at `packages/cortex-core/cortex/assets/brain-source.svg`. User Action runs never depend on Wikimedia uptime. Refresh manually via `scripts/update_brain_anatomy.py`.
- **Python for builders, TypeScript for the 3D viewer** — Three.js is JS-native; Python has the cleaner SVG generation story
- **Schema versioning baked in from day one** — `cortex.yml` has top-level `version: 1`; mismatches raise migration errors with doc links
- **Pydantic = source of truth, JSON Schema = IDE mirror** — both must agree; CI validates examples against both
- **No `Co-Authored-By: Claude` trailers** — disabled in `~/.claude/settings.json`; commits authored solely by the human contributor
- **No telemetry** in the free tier — Pro tier opt-in via `pro.analytics: true`
- **All animations are CSS or SMIL** — never JavaScript (GitHub strips `<script>` from rendered SVGs)
- **Font sizes calibrated for ~0.55× downscale** — README embeds at `width="100%"` on github.com's ~830px content area downscale a 1400px viewBox by ~0.55×; source fonts must be sized accordingly to read at ~13–18px on screen

## Local dev quick reference

```bash
# Install both packages editable
pip install -e packages/cortex-core packages/cortex-cli

# Validate + build
cortex validate examples/standard.yml
cortex build -c examples/extreme.yml -o examples/rendered/extreme/

# Lint + format (must pass CI)
python -m ruff check packages/
python -m ruff format --check packages/

# JSON schema check (must pass CI — same script as .github/workflows/ci.yml)
npm install --no-save ajv ajv-formats js-yaml
node -e 'const Ajv=require("ajv"),addFormats=require("ajv-formats"),yaml=require("js-yaml"),fs=require("fs");const a=new Ajv({allErrors:true,strict:false});addFormats(a);const v=a.compile(JSON.parse(fs.readFileSync("packages/cortex-schema/schema.json","utf8")));for(const n of ["minimal","standard","extreme"]){const ok=v(yaml.load(fs.readFileSync("examples/"+n+".yml","utf8")));console.log((ok?"[OK] ":"[FAIL] ")+n);if(!ok)console.error(JSON.stringify(v.errors,null,2));}'

# Refresh this file's auto-sections
python scripts/refresh_context.py
```

## How this file stays current
- **Auto sections** (between `<!-- AUTO:X:START -->` markers) are regenerated by `scripts/refresh_context.py`, which runs on every push to main via `.github/workflows/refresh-context.yml`. The workflow commits the diff back as `cortex-bot[bot]` with `[skip ci]`.
- **Manual sections** (Active work, Open issues, Design decisions, Code map) are human-edited. Update them in the same PR that changes the underlying state — don't let drift accumulate.
- **The two-file split**: this file = current state (decays fast, auto-friendly). [`VISION.md`](./VISION.md) = future direction (decays slowly, all manual).
